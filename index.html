Goals, Projects, and Tasks

A practical reference guide for George to sort raw ideas into clear goals, projects, and tasks.
Goal / Priority = the business outcome or focus area. Project = the build or initiative. Task = the next action.

1. Why This Guide Exists

  • George has many ideas, notes, and priorities competing for attention.
  • The issue is not lack of ideas. The issue is classification.
  • When goals, projects, and tasks are mixed together, everything feels equally important.
  • This guide helps George sort what is in his head faster and with less mental drag.
$10k/hour rule: George’s time should go toward strategic direction, high-value judgment, and final decisions — not getting stuck sorting low-level tasks.

2. Best Order of Thinking

Step What George Should Do Why It Matters
1 Identify the Goals / Priorities first. This creates the “mother buckets” for the year or planning period.
2 List the projects and tasks George has in mind. This captures the raw ideas without overthinking too early.
3 Classify each item as a project or task. This separates real builds from simple actions.
4 Attach each project or task to its Goal / Priority. This keeps work tied to what actually matters.
Start with the Goals / Priorities first. Then sort projects and tasks underneath them.

3. Core Definitions

Category Definition Answers This Question Examples
Goal / Priority A business outcome or focus area George wants meaningfully advanced. It can be broad, strategic, or not yet ready to execute. What result or business area matters?
  • Wholesale Distribution
  • Loyalty Program + Raffle Program System
  • Inventory Management Science
  • File Management System
Strategic Decision A priority area that needs a direction decision before it can become a clear project. What direction must be chosen first?
  • KISS Studio / Bensalem: operate, lease, repurpose, or exit?
  • Branding: internal development, agency engagement, or phased approach?
Project A temporary, multi-step body of work that creates a defined deliverable in service of a Goal / Priority. What needs to be built, fixed, launched, or completed?
  • Build Lease Register system
  • Create loyalty rewards structure
  • Build local social media campaign plan
  • Create file governance structure
Task A single action with no meaningful sequencing. What is the next action?
  • Upload lease to folder
  • Email broker
  • Rename file
  • Add project to ClickUp

4. Founder Label Rule

If George identifies something as a goal or priority, treat it first as a Goal / Priority Area.

The point is not to debate the label too early. The point is to ask:
  • What does success look like?
  • Does this need a strategic decision first?
  • What projects belong under it?
  • What tasks are only next actions?

5. Simple Working Hierarchy

Level Meaning Example
Goal / Priority Business outcome or focus area George wants advanced. Loyalty Program + Raffle Program System
Strategic Decision Direction that must be chosen before execution is clear. Should KISS Studio / Bensalem be operated, leased, repurposed, or exited?
Project Specific build or initiative that moves the priority forward. Create loyalty rewards structure
Project Tier The amount of structure the project needs. Quarterly / Mini / One-off

6. Fast Classification Questions

Question If Yes If No
Is this a business result or focus area George wants advanced? Goal / Priority Go to next question
Does this need a direction decision before work can start? Strategic Decision Go to next question
Does this require multiple steps, sequencing, ownership, and closure? Project Go to next question
Can this be done as one clear action? Task Clarify the work before classifying
Do not ask only: “Is this a goal or project?”

Ask: “Is this the focus, the decision, the build, or the next action?”

7. Project Tiers

Once something is confirmed as a project, classify the level of rigor needed.

Project Tier Use It When Documentation Level Example
Quarterly Project Multi-step, multi-phase, strategic, high-risk, or founder-heavy. Full Brief Inventory decision-support system
Mini Project Multi-step project needing structure but not full rigor. Lite Brief Build local campaign tracker
One-off Project Short, simple, multi-step project. Brief Description Only Create Google Review automation
Task Single action, no sequencing required. None Add project to ClickUp

8. Practical Examples

Raw Thought Correct Classification Why
“Reduce lease renewal risk” Goal / Priority This is the business outcome.
“Build Lease Register” Project This is the system build that supports the outcome.
“Upload GVO lease” Task This is one action inside the operating process.
“KISS Studio / Bensalem” Goal / Priority Area + Strategic Decision It matters, but direction must be chosen before it becomes a clear project.
“Create raffle mechanics” Project This requires structure, rules, implementation, and review.
“Email agency options” Task This is one next action.

9. What to Avoid

  • Do not turn every task into a project.
  • Do not turn every project into a goal.
  • Do not force a strategic decision into a project before the direction is clear.
  • Do not spend founder time perfecting labels too early.
  • Do not classify projects and tasks first if the Goals / Priorities are not clear yet.

10. Final Shorthand
